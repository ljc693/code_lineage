产品需求文档（PRD）：程序血缘分析中心 V1.0
文档项	内容
版本	V1.0
产品名称	程序血缘分析中心
目标	构建一个自动化的代码级调用关系分析平台，打破研发信息孤岛，为影响评估、变更管理和代码治理提供可视化、可量化的决策依据。
1. 背景与问题
软件加速迭代导致资产信息分散、全链路影响难以评估、代码资产台账逐渐“腐烂”。现有依靠人工梳理和文档记录的方式效率低且不可靠，急需一种自动化、实时化的代码关系分析手段。

2. 用户角色
架构师/技术主管：关注系统全局调用拓扑，识别不合理依赖和架构腐化点。

开发人员：提交代码前了解变更影响范围，查找调用链下游的接口。

测试人员：根据需求变更的影响链路，精准划定测试范围和回归用例。

运维人员：故障发生时，快速定位到异常链路中的源头服务/方法。

系统管理员：维护扫描任务、管理数据规则。

3. 功能需求
3.1 代码资产与血缘采集

多模式扫描：支持全量扫描和增量扫描（基于提交日志的文件变动）。

技术栈支持：覆盖Java项目的源代码和依赖jar包，采用AST（精确提取）与ASM（字节码补全）双引擎。

流水线处理：包含预扫描（识别技术栈）、常规扫描（方法级调用识别入库）、批量数据处理（拼接跨应用链路）三条自动化流水线。

数据保鲜：支持定时（如每日凌晨）或Webhook触发增量扫描，保持血缘关系与代码仓库同步。

3.2 血缘图谱存储与查询

混合存储：关系库存储项目、服务、扫描规则等配置型数据；图数据库（HugeGraph）存储方法、类、服务节点及其CALLS调用边。

统一查询API：提供RESTful接口，支持按服务名、方法签名、HTTP接口路径、交易码等多维度查询完整的上下游调用链。

3.3 血缘族谱可视化

拓扑图视图：以力导向图展示端到端服务调用链路，支持节点展开/收起、路径高亮、全屏与导出。

多视角入口：支持从分布式服务、HTTP入口、定时任务等多种切面进入链路视图。

节点详情：点击任一方法/服务节点，可查看其出入度、所属应用、最近变更记录等元数据。

3.4 交易全链路变动感知

变更映射：获取代码仓库的变更文件列表后，自动在图数据库中匹配受影响的方法节点，并将其在服务链路上进行可视化标注（如红色高亮）。

影响评估报告：生成文本化报告，内容包含：变更文件清单、影响的直接和间接服务、关键受影响方法列表，并支持一键发送给相关责任人。

与需求关联：可手动或自动关联需求ID，实现“需求→变更→影响链路”的闭环追溯。

3.5 程序代码价值识别

关系度量：计算方法节点的出入度（被调/主调次数）和聚合度（簇内调用频率）。

标签自动化：

高影响节点：出度或入度超过阈值（可配置），标识为修改需重点审查。

疑似废弃节点：入度和出度均为0且持续超过N个版本，标记为待评估下线。

关键通道节点：处在多个交易必经路径上的聚合点，标识为链路可用性核心。

列表与视图联动：可在可视化页面筛选查看各类标签节点，并导出清单。

4. 非功能需求
性能：单体应用（约10万方法级）的增量扫描应在5分钟内完成，全链路拼接任务在30分钟内完成。API查询响应时间<500ms。

可扩展性：扫描引擎和存储支持水平扩展，未来可接入其他语言的分析模块。

安全性：代码数据严格权限控制，仅授权角色可查看对应的项目血缘。API需鉴权，敏感代码片段脱敏。

可用性：系统可用性99.9%，支持降级策略（如扫描暂停，查询仍可用）。

5. 数据流程与架构简述
数据流程：

采集：扫描引擎从Git/SVN拉取代码，经过预扫描过滤，再通过AST+ASM解析生成“方法-调用-方法”原始关系。

加工：批量处理流水线消费原始关系，跨应用拼接成“服务-调用-服务”的全链路，并补充交易入口标记。

存储：方法级关系存入图数据库（顶点+边），服务链路以图路径形式沉淀；配置和扫描快照存入关系库。

服务：API服务层从混合存储中查询，组装成全链路JSON或图数据，供前端和外部系统消费。

应用：可视化平台、CI插件、监控告警等消费数据，产生业务价值。

简化架构分层：

业务场景层：可视化门户、影响报告、价值标签、外部系统对接。

集成服务层：标准化API（链路查询、变更标注、统计导出）。

核心数据层：图数据库（血缘关系） + 关系数据库（配置/台账）。

处理引擎：预扫描流水线 → 常规扫描流水线（AST+ASM） → 批量链路拼接。

6. 验收标准
能够准确解析出至少10个试点Java项目的方法级调用关系，精准率≥95%，召回率≥90%。

任意选定一个HTTP接口，系统可还原出从该入口到数据库操作的完整方法调用链，路径断点率<10%。

当提交一次代码变更后，系统能在30分钟内产生一份影响报告，且列出的受影响方法无关键性遗漏。

可视化页面能流畅渲染含500+节点的服务拓扑图，并正确支持高亮和搜索。

代码价值标签的准确率达到人工核对一致性的85%以上。
